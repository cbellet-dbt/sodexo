üèóÔ∏è dbt Platform - The Solution Architecture

1 What is dbt Platform?

dbt Platform is a complete operating system for data transformation:

  • dbt Cloud IDE: Git-connected, cloud-based development environment
  • Smart Orchestration: State-aware (only run what changed)
  • CI/CD Pipeline: Automated testing, staging, approval gates
  • Semantic Layer: Business metrics exposed to analytics tools
  • Observability & Lineage: Auto-generated data catalog with column-level lineage
  • Cost Intelligence: Clear visibility into compute costs by project & team

2 How dbt Solves Sodexo's Challenges

Challenge Today (Pain) dbt Solution
CI/CD Limitations Full runs every commit ‚úÖ Smart CI: test only changed models + downstream
Data Quality Tests run post-processing (too late) ‚úÖ Tests run with every commit (before deployment)
Time-to-Market Complex requirements, rework loops ‚úÖ Automated documentation, lineage visualization
Troubleshooting Manual investigation (hours) ‚úÖ Auto-generated lineage, root cause analysis
Cost Visibility Unknown cost drivers ‚úÖ Cost monitoring, state-aware skip-if-unchanged
Team Scaling Manual processes, training curve ‚úÖ Standardized workflows, knowledge sharing

3 Key Principles of dbt Platform

  • Code-First & Git-Native: All transformation logic versioned on GitHub (transparent, auditable, compliant)
  • Systematic Testing: Quality gates built into development workflow, not post-deployment
  • State-Aware Orchestration: Only recalculate what changed (10-20%+ savings in compute costs)
  • Multi-Warehouse Flexibility: No vendor lock-in; code works on Snowflake, BigQuery, Databricks, etc.
  • Built-in Governance: Lineage, contracts, freshness, testing = trust signals for stakeholders

4 Integration with Sodexo's Current Stack

  • Databricks: dbt natively supports Databricks (Unity Catalog compatible)
  • Azure Data Factory: dbt jobs can be triggered from ADF orchestration (no replacement needed)
  • Power BI: Semantic Layer exposes metrics directly to BI tools
  • Azure ML: Can coordinate data pipelines before ML model training
  • No Rip-and-Replace: Sits on top of existing infrastructure, not replacing it
